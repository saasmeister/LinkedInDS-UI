Never build until the brief is filled. Post-first, type-explicit, brand-explicit. ■ required · □ ask every time

Required

1The post — full text (or the idea)
2Visual type — single / carousel / infographic / quote · ask, don’t guess
3Identity bar — name + claimed category/function
4Brand layer — primary colour + font

Ask every time

Second hook — write it, or derive from post? (differs from the post hook)
Subheading — include one? what does it add?
Signature — underline / block / bubble · once per series
Secondary colour — one colour? offer to derive it for section slides
Type-specific — steps & CTA · the one visual · results stats · the exact quote

Then: a pushback pass — does the post fit this type? does it hold the series together? Build only when the approach is clear. (full spec → BRIEF.md)
